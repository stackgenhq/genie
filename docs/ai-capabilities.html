AI capabilities & Genie

Genie is designed with the DORA AI Capabilities Model in mind — seven practices that amplify the benefits of AI in software development. Below is how we’re aligning with each capability today.

Responsible AI isn’t something one organization defines alone — it takes a village. We’re actively working toward these goals and your participation is key. Where we’re still evolving, we’d love your feedback: what would help your team? What’s missing? Your input shapes how Genie grows.

1. Clear and communicated AI stance

Evolving

Organizations should set clear expectations for AI tool use, support experimentation, and communicate which tools are approved and which actions require approval.

How Genie supports this

  • HITL config — [hitl] with always_allowed and denied_tools lets operators define what the agent can do without approval and what is blocked.
  • Config Builder — Non-technical users can set tool policy via the web Config Builder; docs describe approved vs. denied tools.
  • Chat UX — After connecting, the chat fetches GET /api/v1/capabilities and shows a collapsible “What Genie can do” panel (available tools, auto-approved, blocked). When approval is required, the UI shows why and links to how to add tools to always_allowed.

Where we’re evaluating: Making the “AI stance” more visible to end users up front (e.g. a short “What Genie can do” in docs or first-run). We’d love community feedback: how do you communicate AI expectations in your org?

🗄️

2. Healthy data ecosystems

Evolving

High-quality, easily accessible, and unified internal data significantly amplifies AI’s positive impact on performance.

How Genie supports this

  • Vector store — In-memory or Qdrant for semantic search; skills, documents, and memory are ingested and queryable.
  • Skills — Structured procedures and capability packs loaded via skills_roots and discoverable through the skills system.
  • PII redaction — Configurable redaction before data is stored so sensitive content isn’t persisted in memory.

Where we’re evaluating: Framing skills and memory as the “internal data” that powers Genie and sharing guidance on keeping them healthy. We’d love to hear how you maintain data quality and freshness in your setup.

🔌

3. AI-accessible internal data

Strong

Connecting AI tools to internal data through “context engineering” (RAG, APIs, memory) boosts impact on effectiveness and quality.

How Genie supports this

  • Skills in vector store — Loaded, embedded, and searched so the agent can follow your procedures.
  • Working and episodic memory — Shared context across agent nodes in a run; past accomplishments stored and retrieved for follow-up.
  • MCP — Connect to external data and tools (DBs, APIs, internal services) via the Model Context Protocol.
  • Persona and classification — System prompts and routing so the right context is used for each request.
📂

4. Strong version control practices

Focus area

Frequent commits and robust rollback are especially important as AI increases the volume and velocity of generated code.

How Genie supports this

  • SCM tools — GitHub, GitLab, Bitbucket integrations for repo operations; run_shell can run git commands when allowed.

Where we’re evaluating: Skills and docs guidance on frequent commits and rollback when Genie is used for code or infra — and whether templates that encourage small, revertible commits would help. We’d love community input: what practices work for you?

📦

5. Working in small batches

Evolving

Small batches amplify AI’s positive influence on product performance; the same DORA principle applies in AI-assisted workflows.

How Genie supports this

  • ReAcTree and planning — The planner breaks work into discrete steps; create_agent spawns focused sub-agents with right-sized goals and tool budgets.
  • Persona guidance — “Batch related work,” “Right-size your agents,” and sequence/parallel/fallback flows encourage small, testable units of work.

Where we’re evaluating: Making “small batches” more visible in docs and README so teams and stakeholders can see how ReAcTree and planning support this. We’d love feedback: how do you talk about batching with your teams?

👤

6. User-centric focus

Evolving

Keeping the end-user experience central is critical; adopting AI without this focus can harm teams and outcomes.

How Genie supports this

  • Clarifying questions — The planner uses ask_clarifying_question when intent is ambiguous, avoiding assumptions that hurt users.
  • QA and acceptance — Blackbox acceptance tests in qa/ and AGENTS.md require user-facing validation (Arrange/Act/Assert) for new features.
  • Messenger HITL — Approve or reject actions in Slack, Teams, etc., without leaving the conversation.

Where we’re evaluating: Stating “user-centric focus” more explicitly in our principles and README so it’s clear that Genie is built for end-user outcomes. We’d love to hear how you prioritize user experience when adopting AI tools.

🛤️

7. Quality internal platforms

Strong

Automated, secure “paved roads” let AI benefits scale across the organization and prevent gains from being lost to downstream bottlenecks.

How Genie supports this

  • Skills — Standardized procedures and capability packs so teams don’t reinvent workflows.
  • Toolwrap — Resilience (rate limits, circuit breakers, retries), observability (tracing, metrics), and security (HITL, audit, sanitization) on every tool call.
  • Config-driven behavior — HITL, model routing, and tool policy are configured, not hard-coded, so the same platform can serve different policies.
  • MCP and skills — Extend the “road” to new data sources and capabilities without changing core code.
