# Runbooks — Acceptance Criteria

> Tests for the runbook loader, convention directory discovery, config-driven paths, persona injection, and edge cases.

---

## 25 — Runbook: Convention Directory Auto-Discovery

### Why
Runbooks dropped into `.genie/runbooks/` should be loaded automatically without any configuration.

### Problem
Without convention-over-configuration, users must manually configure every runbook path, increasing onboarding friction.

### Benefit
Zero-config runbook loading — just create the directory and drop files in.

### Arrange
- Create `<project-root>/.genie/runbooks/deploy.md` with content:
```markdown
# Deployment Guide
Always run tests before deploying to production.
```

### Act
```bash
build/genie grant 2>&1 | grep -i runbook
```

### Assert
- Log line: `Runbooks loaded and appended to persona`
- Log line: `runbook loaded` with `path` containing `deploy.md`

---

## 26 — Runbook: Explicit Config Paths

### Why
Users must be able to specify runbook paths outside the convention directory.

### Problem
Some teams store runbooks in shared directories or outside the project root.

### Benefit
Flexible runbook sourcing from any location on the filesystem.

### Arrange
- Create `/tmp/genie-test-runbooks/ops.md` with content:
```markdown
# Ops Runbook
Check pod health before scaling.
```
- Add to `.genie.toml`:
```toml
[runbook]
runbook_paths = ["/tmp/genie-test-runbooks"]
```

### Act
```bash
build/genie grant 2>&1 | grep -i runbook
```

### Assert
- Log line: `runbook loaded` with `path` containing `ops.md`
- The runbook content should be injected into the agent's persona

---

## 27 — Runbook: No Runbooks Found

### Why
When no runbooks exist, the agent should start normally without errors.

### Problem
A missing convention directory or empty config could cause a crash or misleading warnings.

### Benefit
Clean startup for users who haven't configured runbooks yet.

### Arrange
- No `.genie/runbooks/` directory
- No `runbook_paths` in config

### Act
```bash
build/genie grant 2>&1 | grep -i runbook
```

### Assert
- Log line: `No runbooks found` (at DEBUG level)
- No errors or warnings related to runbooks
- Server starts successfully

---

## 28 — Runbook: Unsupported File Types Are Skipped

### Why
Binary files or unsupported formats in the runbook directory must be silently skipped.

### Problem
Attempting to parse binary files could cause panics or corrupt the persona prompt.

### Benefit
Safe, selective file loading that only processes supported formats (.md, .txt, .json, .csv).

### Arrange
- Create `.genie/runbooks/` with:
  - `valid.md` (markdown content)
  - `image.png` (binary file)
  - `data.xlsx` (unsupported format)

### Act
```bash
build/genie grant 2>&1 | grep -i runbook
```

### Assert
- `valid.md` is loaded (appears in logs)
- `image.png` and `data.xlsx` do NOT appear in logs
- No errors or panics
